### Key Design Decisions

1. **Mock STARK Backend:** Python ZK-STARK ecosystem is sparse (winterfell unavailable, cairo-lang too heavy for PoC). Implemented mock backend that demonstrates interface pattern - production can swap in real STARK backend.

2. **Interface Design:**
   - `prove_single_cantor_pair(x, y)` → returns `(result, ZKCantorProof)`
   - `verify_single_cantor_pair(z, proof)` → returns `bool`
   - `ZKCantorProof` dataclass with root, leaf_count, stark_proof, constraint_count

3. **AIR Constraints:** Cantor pairing uses 3 constraints:
   - s = x + y
   - intermediate = s × (s + 1) / 2
   - z = intermediate + y
