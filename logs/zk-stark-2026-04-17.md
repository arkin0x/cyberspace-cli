### Key Decisions Made

1. **STARKs over SNARKs** — No trusted setup required, post-quantum secure. Matches Cyberspace's decentralization values.

2. **Winterfell as primary library** — Rust-based, well-documented, Python bindings available, Meta-backed maintenance. Plonky3 as fallback for GPU support.

3. **Zero-knowledge NOT required** — Leaf values (coordinates, temporal seed) are already public in Nostr events. ZK adds overhead without security benefit. Can use "transparent" STARK variant.

4. **ZK as optimization, not replacement** — Standard verification remains valid. ZK proofs are an *additional* verification path for lightweight clients. Preserves work equivalence property as an option.

5. **Feature flag for initial deployment** — Keep ZK features behind `CYBERSPACE_ZK_ENABLED=1` until production-ready.
