### Key Decisions Made
1. **Library selection: Winterfell** (Facebook/Meta)
   - Mature, production-ready
   - Python bindings for rapid iteration
   - Designed for arithmetic circuits
   - Configurable field size (128-256 bit)
   
2. **Preserve work equivalence**
   - ZK proofs optimize honest verification (fast path)
   - Fraud detection still requires full Cantor recompute (slow path)
   - Prover does MORE work (Cantor + STARK), not less
   - Observer advantage property maintained

3. **Optional feature flag**
   - `cyberspace config set --zk-proofs enabled`
   - New action tag: `A=hop-zk` (backward compatible)
   - Existing `A=hop` remains valid without ZK proof

4. **Implementation approach**
   - Phase 1: Single Cantor pair PoC ( Session 4-8)
   - Phase 2: Full tree with temporal seed (Session 9-15)

### Benchmark Results

**Single Cantor Pair:**
- Prove time: 0.018 ms
- Verify time: 0.009 ms
- Speedup: **2.0x**
- Proof size: **143 bytes**

**Tree Proofs (scaling):**
| Leaves | Height | Prove (ms) | Verify (ms) | Size (KB) | Speedup |
|--------|--------|------------|-------------|-----------|---------|
| 4 | 2 | 0.04 | 0.02 | 0.54 | 2.1x |
| 8 | 3 | 0.05 | 0.03 | 1.22 | 1.6x |
| 16 | 4 | 0.09 | 0.06 | 2.70 | 1.5x |
| 32 | 5 | 0.18 | 0.11 | 5.93 | 1.6x |

**Key insight:** Current implementation is pure Python educational version. Proof sizes are reasonable (~6 KB for 32 leaves), but speedup is modest (1.5-2x) because:
- This is not a full STARK backend (no FRI, no polynomial commitments)
- Both prove and verify use same Python code paths
- Real STARKs (Winterfell, Cairo) would give 100-1000x speedup
