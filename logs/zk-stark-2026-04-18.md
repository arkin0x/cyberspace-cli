### Key Decisions Made

1. **Module Structure:** Integrated with existing `zk_stark/` package instead of creating separate module
   - Existing `circuit.py` already has constraint system implementation
   - My `proof.py` provides the integration layer on top

2. **PoC Approach:** Hash-based commitments instead of full STARK
   - Demonstrates API pattern and integration workflow
   - Placeholder for real STARK library (Winterfell/Cairo)
   - All tests pass with PoC implementation

3. **Verification API:** Returns tuple `(bool, str)` instead of just `bool`
   - Provides status message with timing information
   - Easier debugging and benchmarking

4. **Nostr Tag Design:** Three tags for ZK integration
   - `zk_proof` — Full proof serialized as hex
   - `zk_public_inputs` — Public inputs hash for quick verification
   - `zk_scheme` — Proof scheme identifier (e.g., "winterfell-stark-v1")
